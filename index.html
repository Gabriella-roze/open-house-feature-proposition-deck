WHY NOW

The opportunity is product-led—not rent-only.

The strongest case is not a collection of broad market statistics. It is a clear product gap: scheduled viewings already happen, but they are not consistently structured, discoverable or sold as a listing advantage.

Existing behaviour

Agents already group prospects into shared viewing windows. The feature standardises that workflow instead of creating a new habit.

Product hypothesis from the proposed Greek call-back workflow.
Existing lead rails

Spitogatos introduced Book a Visit in June 2025, giving Open House a familiar request and agent-management foundation.

Source · Spitogatos
Full-market scope

Spitogatos already spans sale and rent across homes, commercial property, land and new developments. Open House can be enabled wherever an agent wants to create an in-person event.

Source · Spitogatos
Clear paid value

Only paid-enabled listings receive the event module, badge and filter eligibility. That makes the agent benefit visible before any complex pricing model is introduced.

Proposed product rule—not an existing Spitogatos policy.
STRATEGIC IMPLICATION

Open House should be positioned as a paid event layer for any agent listing: discoverability, batch coordination and a registrant list in one simple add-on.

AGENT VALUE

The feature is worth paying for when it saves time and changes listing visibility.

Open House is not payment for another contact button. It is payment for a discoverable event, batch coordination and a named follow-up list.

Batch appointments

Bring several prospects to one planned window instead of arranging repeated private visits.

Listing differentiation

Only enabled listings qualify for the Open House filter and badge. Among duplicate property ads, the enabled agent can gain the event-specific surface.

Higher-intent requests

The user chooses a real date and time before sending contact details, giving the agent a clearer reason to follow up.

Agent control

Set capacity, call back, confirm, waitlist or cancel. The platform organises the event without replacing the local relationship.

Portfolio-wide reuse

Use the same paid add-on for sale, rent, commercial property or a new-development viewing event.

Measurable outcomes

Track requests, confirmations, attendance and follow-up so the office can judge whether the activation paid back.

KEY RISKS

The main risks are commercial and operational—not technical.

The pilot should treat event supply, no-shows, adoption, duplicate listings and billing clarity as first-class product problems.

01

No-shows

Call-back confirmation, reminders, waitlist and attendance status.

02

Empty filter

Seed enough events by area, category and day; provide a nearby-date fallback.

03

Lead cannibalisation

Cross-channel dedupe, matched holdout and net-revenue reporting.

04

Weak paid adoption

Make the entitlement concrete: badge, filter, registrant list and outcome reporting.

05

Privacy & address

Reveal policy, minimum data, retention rules and consented notifications.

06

Duplicate property listings

Clarify that filter visibility belongs to each paid-enabled listing, while preventing duplicate events from the same office.
